# Sync API Schema Skill

This skill explains the workflow for maintaining the backend and frontend API types in sync using OpenAPI (`swagger`) and Orval.

## How it works

The monorepo uses a shared directory `schema/` at the root which acts as the source of truth for the API contract between the backend and frontend.

### 1. Backend Generation (`schema/openapi.json`)
The backend is configured to automatically write the current API schema to the shared folder on startup.
- **Location**: `backend/src/main.ts`
- **Condition**: It only executes when `NODE_ENV !== 'production'`.
- **Output**: Writes `openapi.json` into the root `schema/` folder.
- **Action**: A developer only needs to run `pnpm run start:dev` in the `backend/`, and the schema is automatically updated whenever they save changes to Controllers or DTOs.

### 2. Frontend Consumption
The frontend utilizes `orval` to read the shared `openapi.json` and generate fully-typed React Query hooks/Axios clients.
- **Configuration**: `frontend/orval.config.ts` handles the splitting of tags into separate domains (e.g., Identity, Notifications) and links the output to the custom `api-client.ts` Axios instance.
- **Output Location**: Generated types and endpoints reside in `frontend/src/api/generated/`.

## Developer Workflow

Whenever you modify a Controller or DTO in the backend that affects the API:

1. Ensure the backend is running or restart it (`pnpm run start:dev` in `backend/`). This instantly updates `../schema/openapi.json`.
2. Move to the frontend directory (`cd frontend`).
3. Run the generation script: `bun run generate-types`.
   *This command cleans the old generated API directory using `rimraf` and runs `orval` to reconstruct it.*
4. The frontend is now fully synced with the latest backend types and endpoints.

### Using the Generated Functions
Once generated, you can directly import the HTTP client functions into your Next.js components or hooks. For example:
```typescript
import { usersControllerCreate } from '@/src/api/generated/identity/identity/identity';

// Usage:
const response = await usersControllerCreate({ name: 'John Doe', email: '...', password: '...' });
```
*(Remember to use `'use client';` if you are using hooks like `useTranslation` or React State alongside these calls!)*
